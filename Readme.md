## &nbsp;вңҰ &nbsp;What is Readwise?

**Readwise** is an AI-powered PDF chat assistant built with Streamlit and OpenRouter. Upload any PDF вҖ” research paper, contract, textbook, report вҖ” and have a real conversation with it. No summaries that miss the point. No copy-pasting into ChatGPT. Just ask, and get precise answers grounded in your document.

> *Built in a weekend. Runs on free models. Actually works.*

## &nbsp;рҹҡҖ &nbsp;Getting Started

### 1 &nbsp;вҖ”&nbsp; Clone the repo

```bash
git clone https://github.com/Anikett77/readwise.git
cd readwise
```

### 2 &nbsp;вҖ”&nbsp; Install dependencies

```bash
pip install -r requirements.txt
```

### 3 &nbsp;вҖ”&nbsp; Get your free API key

1. Go to **[openrouter.ai](https://openrouter.ai)**
2. Sign up вҶ’ API Keys вҶ’ **Create Key**
3. Free credits on signup, no billing required

### 4 &nbsp;вҖ”&nbsp; Set up your environment

Create a `.env` file in the root directory:

```env
OPENROUTER_API_KEY=sk-or-v1-your-key-here
```

### 5 &nbsp;вҖ”&nbsp; Run it

```bash
streamlit run RAG.py
```

Open **http://localhost:8501** and you're good to go. рҹҺү

## &nbsp;рҹӣ пёҸ &nbsp;How It Works

```
  User uploads PDF
        в”Ӯ
        в–ј
  pypdf extracts full text
        в”Ӯ
        в–ј
  Text injected into system prompt
        в”Ӯ
        в–ј
  User asks a question
        в”Ӯ
        в–ј
  OpenRouter API called with model of choice
        в”Ӯ
        в”ңв”Җв”Җ 200 OK вҶ’ return answer вң“
        в”Ӯ
        в””в”Җв”Җ 429 / 404 вҶ’ auto-fallback to next model вҶ’ retry
```

Every message in the conversation carries the **full document context** вҖ” so follow-up questions work perfectly without re-uploading.

## &nbsp;вҡ пёҸ &nbsp;Limitations

- Works best with **text-based PDFs** (not scanned image PDFs without OCR)
- Free tier models may occasionally be **slow or rate-limited** вҖ” auto-fallback handles this
- Very large PDFs are **truncated at ~60,000 characters** to fit model context windows
